# 학생 설문 (구글 폼 대체) 설정

학생은 `respond.html?s=설문ID` 에서 **명렬표 기준 반·번호**를 고른 뒤 설문에 응답합니다.  
응답은 **고정 열 구조**의 스프레드시트에 쌓여, 결과 조회(`index.html`)에서 반별로 볼 수 있습니다.

## 1. Apps Script 배포 (필수 — 다른 기기·학생 참여)

1. [응답용 스프레드시트](https://docs.google.com/spreadsheets/d/1oH3Er_9UF_A6HDQEK_1KjFxp54M_JJrU)를 열거나 새 시트를 만듭니다.
2. **확장 프로그램 → Apps Script** 에 `scripts/google-apps-script/survey-api.gs` 내용을 **전체 붙여넣기** 합니다.
3. **배포 → 새 배포 → 웹 앱**
   - 실행: 나
   - 액세스: **모든 사용자**(익명 포함) — 학생·교사 PC에서 제출 가능
4. 배포 URL (`https://script.google.com/macros/s/…/exec`) 을 복사합니다.
5. 기존에 배포한 적이 있다면 **새 버전으로 재배포**해야 `surveyStatus`(진행/완료) 반영이 됩니다.

## 2. 관리자에서 설문 등록

1. `admin-login.html` 로그인 후 **`admin-survey.html`**
2. **Apps Script 웹 앱 URL** 입력 (한 번만 저장됨)
3. 설문 이름, **설문 상태**(진행중/완료), 문항 작성 → **메인에 등록**
4. **서버(Config 시트) 동기화 완료** 메시지 확인
5. **학생 참여 링크** 복사 후 공유 (`&roster=` 포함 시 명렬표 시트도 함께 전달)

### 설문 상태

| 상태 | 학생 참여 | 메인 화면 |
|------|-----------|-----------|
| **진행중** | 참여 링크로 응답 가능 | 진행 안내·미리보기 |
| **완료** | 참여 차단 (서버에서도 거부) | 버튼 클릭 → 분석 결과 |

상태 변경: `admin.html` 목록의 **진행중/완료** 선택 또는 `admin-survey.html` 저장 시 선택.

## 3. 명렬표

`admin.html` → **명렬표 입력** → 구글 시트 URL, **다음에도 기억** 체크.  
시트 공유: **링크가 있는 모든 사용자 → 보기**.

## 4. 결과 조회

- **완료된 설문**: 메인 `index.html` → 완료 섹션에서 설문 클릭 → 반별 분석
- **진행중 설문**: 메인에서 미리보기 가능, 학생은 참여 링크만 사용

응답·Config 시트 공유: **링크가 있는 모든 사용자 → 보기** (gviz 조회용).

## 데이터 구조

| 시트 | 열 |
|------|-----|
| Config | id, label, grade, questionsJson, categorySelectAll, **surveyStatus** |
| Responses | 제출시각, 설문ID, 학번, 반, 번호, 이름, 문항별 열 |

## 체크리스트 (참여 링크 오류 시)

- [ ] 웹 앱 URL이 `…/exec` 로 끝나는지
- [ ] 설문 저장 후 **서버 동기화 완료** 되었는지
- [ ] Apps Script **재배포** 후 최신 `survey-api.gs` 인지
- [ ] 명렬표·응답 시트가 **보기** 공유인지
- [ ] 완료 설문은 학생이 응답할 수 없음 (의도된 동작)

## 제한

- 관리자 PC의 `localStorage`에도 설문 목록이 저장됩니다. 다른 PC 관리 시 **웹 앱 동기화**로 Config 시트를 기준으로 합니다.
- 제출·원격 설문 조회는 **웹 앱 URL + 공개 시트** 가 필요합니다.
- **학번+설문ID** 기준으로 이미 제출한 학생은 다시 제출할 수 없습니다. 수정이 필요하면 Responses 시트에서 해당 행을 삭제한 뒤 재제출하세요.
- 설문 문항을 수정해 응답 시트 열(헤더)이 바뀌어도 **기존 응답 행은 유지**됩니다. 새 문항 열은 비어 있고, 삭제된 문항 열 값은 더 이상 표시되지 않습니다.
